Story

현상 유지하지 않은 5년 6개월의 기록

Chapter 01

웹/앱 접근성 마크 획득

한국정보접근성인증평가원 심사를 앞두고, 운영 중인 서비스에 영향을 주지 않으면서 마크업과 구조를 전면 점검해야 했다. 심사 대응, 직접 마크업 수정과 동시에 내부 가이드를 작성해 이후 작업의 기준도 만들었다. 결과적으로 웹·앱 접근성 인증 마크를 획득했다.

Chapter 02

배포 단계 안내 & CBT 환경 분리

퍼블리셔 산출물과 JSP의 배포 순서가 명확하지 않아 혼선이 생기는 상황이었다. 시스템 흐름을 파악한 뒤 담당자를 설득해 배포 순서를 정리했다. 또한 기존에는 CBT 환경 구축이 미비했으나, CSS·JS 파일을 클론해 운영 서버 안에서 CBT 환경을 분리함으로써, 안정적인 CBT 테스트가 가능해졌다.

Chapter 03

CSS → SCSS 마이그레이션

레거시·더미 코드가 누적되고 네이밍 기준이 없는 상태에서, 코드 간 간섭으로 하나를 수정하면 다른 곳에서 문제가 생기는 일이 반복됐다. CSS를 SCSS로 마이그레이션하고 7-1 패턴을 도입해 역할별로 파일을 분리했다. 네이밍 규칙도 이때 함께 정립해, 코드 간 간섭이 사라지고 신규 기능 추가나 스타일 수정이 예측 가능한 구조가 됐다.

Chapter 04

JS 모듈화

프론트엔드 전반에서 공통 기능이 제각각 구현되어 있었다. show, hide, 아코디언, 스켈레톤 같은 반복 패턴을 공통 모듈로 작성하고 export/import로 관리했다. 호출은 코드 한 줄, 옵션은 객체로 전달하는 구조로 설계해 누구든 쉽게 가져다 쓸 수 있게 했다.

  var idx = $(this).index();
  $('.tab-btn').removeClass('is-active');

});

// 모달 — 또 중복
$('.modal-open').on('click', function() {

코딩 가이드

SCSS

중첩 최대 3단

과도한 특이성(specificity) 방지. 유지보수를 위해 깊은 중첩을 지양. 

.header-nav {
  &__list { ... }
  &__item {
    a { color: red; } // ✓ 3단
  }
}
SCSS

속성 작성 순서 통일

가독성을 위해 순서를 고정. @include → 레이아웃 → 타이포 → 배경 → 기타. 

.element {
  @include flex(center); // 1. mixin
  display: flex;         // 2. 레이아웃
  font-size: 14px;       // 3. 타이포
  background: #fff;      // 4. 배경
  opacity: 0.8;          // 5. 기타
}
JS

선택자 캐싱 필수

반복 DOM 탐색 방지. jQuery 객체는 $ 접두사로 일반 변수와 구분. 

// Bad
$('.menu').show();
$('.menu').css('color', 'red');

// Good
const $menu = $('.menu');
$menu.show();
$menu.css('color', 'red');
HTML

상태 클래스 분리

BEM Modifier(--)는 영구적 변형, State Class(is-/has-)는 일시적 상태. 

/* BEM — 생김새 변형 */
.btn--primary { }

/* State — 동작 상태 */
.btn.is-disabled { }
.input.is-error { }
.list.has-data { }

주석 원칙

코드가 "무엇을"이 아닌 "왜 이렇게 짰는지"를 설명. HTML은 S:/E: 영역 구분. 

<!-- S: 로그인 폼 영역 -->
...
<!-- E: 로그인 폼 영역 -->

// Bad:  버튼을 숨긴다
// Good: 약관 동의 전에는 노출 불필요
